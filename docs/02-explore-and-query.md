# Explore And Query

## Query And Data

Route:
- `/data`

Purpose:
- Main data workspace for table browsing, schema inspection, and SQL work.

Typical workflow:
- Select connection.
- Choose database or schema.
- Browse tables or run SQL.
- Inspect results and export when needed.

## Saved Queries

Route:
- `/saved-queries`

Purpose:
- Stores reusable SQL for repeat work.

Primary actions:
- Create query.
- Organize and reopen saved query entries.
- Re-run saved SQL against a target connection.

Screenshot:
- `docs/screenshots/saved-queries-page.png`

## ER Diagram

Route:
- `/er`

Purpose:
- Visualizes table relationships and database structure.

Primary actions:
- Inspect entity relationships.
- Navigate between tables visually.
