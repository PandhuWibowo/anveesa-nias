# Admin And Governance

## Connections

Route:
- `/connections`

Purpose:
- Manages database connection definitions and environments.

Primary actions:
- Create connection.
- Edit target host, driver, and environment.
- Validate connection details.

## Approval Workflows

Route:
- `/workflows`

Purpose:
- Configures approval routing by connection and access group.

Primary actions:
- Create workflow.
- Assign connections.
- Assign groups.
- Configure approval steps and approvers.
